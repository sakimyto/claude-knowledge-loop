## Design Principles

- **Schema-first development** — Invest time in data model design; UI/app layers can be regenerated by LLMs when the schema is right
- **Feedback loop over perfection** — Ship, measure, learn, iterate. A fast loop beats a perfect first attempt
- **Convention over configuration** — Reduce decisions by establishing patterns; deviate only with documented rationale
- **Offline-first architecture** — Core functionality must work without network; sync when connected

## Business Rules

- **Content approval required** — All public-facing content needs human review before publishing; no fully automated posting
- **90-day data retention** — User behavioral data older than 90 days must be aggregated or deleted per privacy policy
- **Pricing display rules** — Always show tax-inclusive price in JP market; tax-exclusive with note in US market

## Technical Decisions

- **Bun over Node for scripts** — 3x faster execution, native TypeScript; adopted after Bun 1.2 stabilized (2025-06)
- **SQLite for local data** — Eliminates network dependency; cr-sqlite enables multi-device sync without a server
- **Biome over ESLint+Prettier** — Single tool for lint+format, 10x faster, simpler config; migrated 2025-08

## Quality Criteria

- **Error messages are actionable** — Every error must tell the user what went wrong AND what to do next
- **Tests read as specification** — Test names describe behavior, not implementation; no comments needed
- **No dead code** — Remove unused code immediately; git history preserves it if needed later
- **One-line entries in knowledge docs** — Each knowledge item is self-contained in a single line for LLM readability
